당신은 15년 경력의 시니어 풀스택 개발자이자 머신러닝 엔지니어입니다.
지금부터 모바일 웹 + 서버 추론 방식의 스마트포장 칩 색상 판별 시스템을 개발하세요.

목표는 모바일 브라우저에서 열리면 즉시 카메라 모드로 시작하고, 포장 칩(지시계)이 화면에서 인식되면 자동으로 캡처 및 서버 추론을 수행하여, 결과를 “매우 신선”, “섭취 가능”, “섭취 비권장” 중 하나로 표시하는 웹앱을 만드는 것입니다.

1. 개발 목표

다음 요구사항을 만족하는 실행 가능한 전체 프로젝트를 만들어 주세요.

필수 기능
사용자가 모바일 브라우저에서 URL을 열면 즉시 카메라 화면이 표시되어야 함
후면 카메라를 우선 사용
화면 중앙에 칩(지시계) 인식용 가이드 프레임(ROI 박스) 표시
사용자가 칩을 프레임 안에 위치시키면,
프레임 내 이미지를 자동 분석 대상으로 사용
일정 조건 충족 시 자동 캡처
서버로 이미지 전송
ResNet50 기반 추론 수행
추론 결과 화면에는 아래를 표시
판별 결과: 매우 신선 / 섭취 가능 / 섭취 비권장
하단에 판별 이유
하단 버튼 2개: 종료 / 재판별
재판별 클릭 시 다시 첫 카메라 화면으로 복귀
종료 클릭 시
먼저 window.close() 시도
실패하면 전체 화면에 “브라우저를 닫아 주세요” 또는 “이전 화면으로 이동합니다” 안내 표시
가능하면 history.back()도 시도
즉, 모바일 웹 보안 정책을 고려한 graceful fallback 구현
전체 UI는 모바일 우선으로 설계
반응 속도는 최대한 빠르게 구현
카메라 프리뷰, 분석 상태, 결과 화면 전환이 부드럽게 동작해야 함
2. 결과 상태 정의

모델 추론 결과는 내부적으로 아래 3개 클래스로 처리하세요.

very_fresh → 화면 표시: 매우 신선
consumable → 화면 표시: 섭취 가능
not_recommended → 화면 표시: 섭취 비권장
이유 문구 예시

각 결과에 대해 하단 이유를 함께 표시하세요.

매우 신선:
"칩의 색상 패턴이 신선 상태 기준과 매우 유사합니다."
섭취 가능:
"칩의 색상 패턴이 섭취 가능한 범위에 해당합니다."
섭취 비권장:
"칩의 색상 패턴이 품질 저하 또는 변질 가능성을 나타냅니다."

이 문구는 상수로 관리하고, 추후 쉽게 변경 가능하게 하세요.
